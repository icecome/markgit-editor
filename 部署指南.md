## Docker 部署

### 1. Docker Compose（推荐）

创建 `docker-compose.yml`：

```yaml
version: '3.8'
services:
  markgit-editor:
    build: .
    ports:
      - "13131:13131"
    environment:
      - PRODUCTION=true
      - GITHUB_CLIENT_ID=${GITHUB_CLIENT_ID}
      - GITHUB_CLIENT_SECRET=${GITHUB_CLIENT_SECRET}
      - CORS_ORIGINS=${CORS_ORIGINS}
      - REDIS_URL=redis://redis:6379/0
    volumes:
      - markgit-cache:/app/blog_cache
    depends_on:
      - redis
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    volumes:
      - redis-data:/data
    restart: unless-stopped

volumes:
  markgit-cache:
  redis-data:
```

启动服务：

```bash
# 创建 .env 文件
cp .env.example .env
# 编辑 .env 配置

# 启动
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 2. 单独 Docker 运行

```bash
# 构建镜像
docker build -t markgit-editor .

# 运行容器
docker run -d \
  --name markgit-editor \
  -p 13131:13131 \
  --env-file .env \
  -v markgit-cache:/app/blog_cache \
  markgit-editor
```

## 传统服务器部署

### 1. 系统要求

- Python 3.11+
- Git
- Redis（可选）

### 2. 安装步骤

```bash
# 克隆项目
git clone https://github.com/your-repo/markgit-editor.git
cd markgit-editor

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 启动服务
python main.py
```

### 3. 生产环境配置

使用 Gunicorn + Uvicorn：

```bash
pip install gunicorn

gunicorn main:app \
  -w 4 \
  -k uvicorn.workers.UvicornWorker \
  -b 0.0.0.0:13131 \
  --access-logfile - \
  --error-logfile -
```

### 4. Systemd 服务

创建 `/etc/systemd/system/markgit-editor.service`：

```ini
[Unit]
Description=MarkGit Editor
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/markgit-editor
Environment="PATH=/opt/markgit-editor/venv/bin"
EnvironmentFile=/opt/markgit-editor/.env
ExecStart=/opt/markgit-editor/venv/bin/gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 127.0.0.1:13131
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
systemctl daemon-reload
systemctl enable markgit-editor
systemctl start markgit-editor
```

### 5. Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    # 强制 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    # 安全头
    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;
    add_header X-XSS-Protection "1; mode=block";
    
    location / {
        proxy_pass http://127.0.0.1:13131;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## PaaS 平台部署

### Railway（推荐）

1. 安装 CLI：`npm install -g @railway/cli`
2. 登录：`railway login`
3. 初始化：`railway init`
4. 添加 Redis：`railway add --plugin redis`
5. 设置环境变量：
   ```bash
   railway variables set GITHUB_CLIENT_ID=your_id
   railway variables set GITHUB_CLIENT_SECRET=your_secret
   railway variables set CORS_ORIGINS=https://your-app.up.railway.app
   railway variables set PRODUCTION=true
   ```
6. 部署：`railway up`

### Fly.io

1. 安装 CLI：`curl -L https://fly.io/install.sh | sh`
2. 登录：`fly auth login`
3. 创建应用：`fly apps create markgit-editor`
4. 创建存储：`fly volumes create markgit_data --size 1`
5. 设置密钥：
   ```bash
   fly secrets set GITHUB_CLIENT_ID=your_id
   fly secrets set GITHUB_CLIENT_SECRET=your_secret
   fly secrets set CORS_ORIGINS=https://markgit-editor.fly.dev
   ```
6. 部署：`fly deploy`

### Render

1. 连接 GitHub 仓库
2. 创建 Web Service，选择 Docker 环境
3. 配置环境变量
4. 添加 Redis 服务

### Zeabur

1. 访问 zeabur.com
2. 连接 GitHub 仓库
3. 选择项目并部署
4. 添加 Redis 服务
5. 配置环境变量

## 环境变量配置

### 必需变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `GITHUB_CLIENT_ID` | GitHub OAuth 客户端 ID | `Ov23li...` |
| `GITHUB_CLIENT_SECRET` | GitHub OAuth 客户端密钥 | `eb848b...` |

### 生产环境必需

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `CORS_ORIGINS` | 允许的来源 | `https://your-domain.com` |
| `PRODUCTION` | 生产环境标志 | `true` |

### 可选变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `PORT` | `13131` | 服务端口 |
| `REDIS_URL` | - | Redis 连接地址 |
| `BLOG_CACHE_PATH` | `./blog_cache` | 缓存目录 |
| `SESSION_TIMEOUT_HOURS` | `1` | 会话超时时间 |
| `MAX_CONCURRENT_SESSIONS` | `100` | 最大并发会话数 |
| `SSL_VERIFY` | 生产 `true` | SSL 证书验证 |
| `GIT_OPERATION_TIMEOUT` | `120` | Git 操作超时（秒） |
| `GIT_CLONE_TIMEOUT` | `300` | Git 克隆超时（秒） |
| `MAX_FILE_CONTENT_SIZE` | `1048576` | 文件最大长度（字节） |

## 常见问题

### Q1: Git 操作失败

**解决方案**:
1. 确保用户已通过 OAuth 登录
2. 检查 OAuth token 是否有效
3. 确保仓库权限正确

### Q2: 会话丢失

**解决方案**:
1. 配置 Redis 存储
2. 检查 `SESSION_TIMEOUT_HOURS` 设置

### Q3: CORS 错误

**解决方案**:
1. 检查 `CORS_ORIGINS` 配置
2. 确保包含前端域名
3. 生产环境必须配置正确的域名

### Q4: SSL 证书错误

**解决方案**:
1. 开发环境可设置 `SSL_VERIFY=false`
2. 生产环境确保证书有效
